lesson1_网络安全各方向介绍
2023/10/18

主要就业发展方向

web 安全

这个方向更符合于大部分人对“黑客”的认知，他们能够黑手机、黑电脑、黑网站、黑服务器、黑内网，万物皆可黑。
相比二进制安全方向，这个方向初期更容易入门一些，掌握一些基本技术，拿起各种现成的工具就可以开黑了。
不过，要想从脚本小子变成黑客大神，这个方向越往后，需要学习和掌握的东西就多了。

比较直白的一个定义是，Web 安全是包括网站、网络应用、网络服务在内的一系列安全内容，或者说，Web 安全关注的是应用层的安全，尤其是细化到与网络相关内容的安全。Web 安全需要学习的内容主要包括网络协议本身、网络应用的特性与对应的安全问题、各种工具的使用。

学会一定的基础之后自己去尝试着挖挖 SRC，俗称挖洞。

就业岗位多为：红队、安全研究、安服、代码审计、安全开发等

二进制安全

二进制安全方向，这是安全领域几大技术方向之一。
这个方向主要涉及到软件漏洞挖掘、逆向工程、病毒木马分析、病毒免杀等工作，涉及操作系统内核分析、调试与反调试、反病毒等技术。因为经常都是与二进制的数据打交道，所以久而久之用二进制安全来统称这个方向。

这个方向的特点是：需要耐得住寂寞。
比不上安全研发可以有实实在在的产品输出，也比不上网络渗透方向听起来的狂拽炫酷，这个方向更多时间是在默默的分析和研究。
以软件漏洞挖掘为例，光是学习五花八门的攻击手法就需要花不少的时间。在这个领域，为了研究一个问题，可能花费数月甚至数年时间，一般人不容易坚持下来。不仅如此，不是勤奋就可以成功，更多还要靠天分。

像腾讯几大安全实验室的掌门人，业界知名的 TK 教主、吴石这些人物，他们已经深谙漏洞挖掘的奥义，并将这门绝技融会贯通，做个梦都能想到新的玩法。不过像这样的天才实在是少之又少，绝大多数人都无法企及。

主要竞赛形式

CTF

网络安全竞赛中最常见的方式，CTF（Capture The Flag，夺旗赛）起源于 1996 年 DEFCON 全球黑客大会，是网络安全爱好者之间的竞技游戏。
CTF 竞赛中又细分方向，如 web 手、reverse 手、misc 手等，如同 ACM 比赛中也分数据结构手、DP 手等。绝大多数比赛的初赛、部分半决赛、决赛为这种形式，大多数情况下为线上赛。
几个小众的方向详细介绍见CTF WIKI

AWD

部分比赛的半决赛、决赛使用这种方式，90%以上为线下赛。意为真实攻防，接近真实攻防场景，强调实战性、实时性、对抗性，综合考量竞赛队的渗透能力和防护能力。
比赛时一般一个队分一台或多台主机和服务，攻击开始前为各个队伍审计代码，查找修补漏洞的时间，攻击开始后 15 分钟生成一轮 flag，攻破其他队伍的服务拿到 flag 提交即可得分，如果该服务的漏洞一直没有修补则攻击队伍可以一直拿到 flag，直到该服务被修补为止。

知识竞赛

多为水赛，一般为线上赛，题目类型多样，包括但不限于选择题、填空题、判断题、简答题、编程题等，一般为 CTF 比赛的补充，拿奖会发证书，有些有奖金，反正也不怎么费时间，不拿白不拿。

学习网络安全的注意事项

  • 在基本上了解了这个行业之后，最好能找到自己感兴趣的方向，然后在这个方向上深入学习，技术类社团最好只专注于一个，这样才能有所建树，比较忌讳什么都会点，什么都不精。

  • 建议略读一下《中华人民共和国网络安全法》，起码知道哪些行为比较危险。不要自作聪明，不要小看晶哥的能力，尽量不要在未授权的情况下对国内网站进行非法渗透，真出事了你挂多少层代理，做多少反侦察晶哥都能分分钟找到你本人。

  • 多实战，实战可以增进你打竞赛的能力，但是只打竞赛却很难增进你的实战能力和工作能力。

  • 测试学校网站的时候不要上大型/重型扫描器 批量打poc等可能会让服务直接挂掉的工具，更不要进行 DDoS、CC 攻击，这是毫无意义也是违法的行为。

  • 多看文档，多看源码，多用Google、Github、StackOverflow、linux社区等，看不懂就开翻译，尽快舍弃掉百度、csdn（有时候还是有点用）等国产垃圾平台

  • 如果想要系统的打下特别扎实的基础，你需要在三年内学好最少一门编程语言、操作系统、计算机网络、计算机组成原理等课程，linux 要基本了解常用命令和文件系统，会用bash或者python、js等脚本语言写脚本，起码要能读懂。英语好的同学可以直接参考CSDIY学习这几门课程。

  • 如果想要边实操边学习，你学到的知识点可能很碎，记得及时总结，否则不出一周你就忘得一干二净。

以后常用的一些缩写解释

PoC，全称“Proof of Concept”，中文“概念验证”，常指一段漏洞证明的代码。

Exp，全称“Exploit”，中文“利用”，指利用系统漏洞进行攻击的动作。

Payload，中文“有效载荷”，指成功 exploit 之后，真正在目标系统执行的代码或指令。

Shellcode，简单翻译“shell 代码”，是 Payload 的一种，由于其建立正向/反向 shell 而得名。

CVE，公共漏洞和暴露（英语：CVE, Common Vulnerabilities and Exposures）又称通用漏洞披露、常见漏洞与披露，是一个与信息安全有关的数据库，收集各种信息安全弱点及漏洞并给予编号以便于公众查阅。此数据库现由美国非营利组织 MITRE 所属的 National Cybersecurity FFRDC 所营运维护。我们经常会在更新电脑系统时看到“修复了CVE-2023-27372等漏洞”类似的字眼，此处指的便是CVE下发的漏洞编号，同时也可以向该组织提交漏洞，通过后会获得你专属的CVE编号，含金量较高。

RCE，任意代码执行（简称 ACE）是指攻击者能够让目标电脑或目标进程中执行任意命令或代码[1]。如果系统有地方可以被黑客利用以执行任意代码，则此处被称为任意代码执行漏洞。特别设计利用此一漏洞的程序，称为任意代码执行漏洞利用。可以通过网络（尤其是通过互联网等广域网）让目标电脑（远程电脑）执行任意代码的能力称为远程代码执行（RCE）。

SRC，Security Response Center即应急响应中心，提交你挖到的漏洞的地方，会发放证书或者奖金

lesson0_工作室介绍
2023/10/18

基本信息

大连东软信息学院网络安全工作室(Neusoft Security Studio,简称NSS)，位于软件园校区A3-200，是由大东软的网络安全爱好者们和网络信息中心共同发起并组成的一个全校性的群众团体。成立的目的是提供一个供爱好者们互相交流学习的平台。

社团氛围自由开放，没有上下级制度，没有官僚体制，不设门槛，一切以交流长进技术为最终目的，是与ACM，微光并行的学校三大技术社团之一。



（工作室内电子垃圾众多，包括几台废弃主机、交换机、软路由、机架式服务器、电焊、热风枪、各种开发板等等，喜欢折腾的同学应该会玩的开心）

特色活动

  • 每周小聚

每周，通常是周六晚上，在工作室/工作室旁边的教室举行每周小聚，会同时开启钉钉会议，使得博川校区的同学们也能够参与进来，内容主要是同学们自由分享最近学到的新技术、新知识。不限制分享内容，最好是网络安全相关，但是运维、开发、硬件等计算机相关方面的也欢迎。

  • 网络安全竞赛

一年大约10次左右的竞赛参赛机会,包含市级省级国家级和各个难度梯度以及各种比赛形式的比赛，自由组队，自由决定是否参加，可以申请学校加分项目/课程加分，基本都是一次性加满，是很好的实践学到的理论知识的机会，部分比赛获奖会有奖金，上限可达数万元/次。

  • 网络安全宣讲

本校老师一直致力于申请网络空间安全专业但未果，目前我校一部分专业已经开始教授网络安全方面的知识，届时会有一些程度较好的同学受到系里邀请，作为特邀讲师为同学们科普和讲授一些网络安全方面的知识。

  • 团建

基本上一年一次，组织聚餐和出游，也有机会和更多的业内大佬进行交流，增强技术，扩展人脉，不强制参加，费用视社团经费情况报销。

  • 授权行动

略

考核形式

时间一般在下半学期开学之后的一段时间，考题为CTF+靶机，做出50%以上即可通过考核成为正式成员，题目均为入门和简单难度，只要跟上进度就能通过，不要太有心理压力。

基础稍差的同学也会视进步情况决定是否通过。

成为工作室正式成员后你能获得的

  • 加入核心小群，跟大佬面对面交流

  • 各安全大厂的内推，包括但不限于奇安信、绿盟、深信服、暗泉等等

  • 使用工作室内部丰富的资源，前沿的网安资料，个人服务器，wiki，云储存，最新的漏洞exp，poc，各种0day

  • 工作室钥匙

工作室内注意事项

  • 获得比赛奖金、通过工作室介绍的项目/短期工作，要拿出5%当做社团经费，上限为500元，经费账目完全公开，可以对经费的用途提出质疑。经费用作工作室日常物件维护、团建等

  • 禁止在工作室内抽烟

  • 注意用电安全，不要损坏公共财产

  • 工作室有门锁，来之前可以问一问工作室有没有人

  • 书架上陈列的书可以任意借阅观看，但是带走要告知一声

  • 新入社的同学阅读一下提问的智慧，提高交流效率。

Fedora-RISCV-Installing(Unofficial)

Boot under QEMU

#Download disk image
wget https://dl.fedoraproject.org/pub/alt/risc-v/disk_images/Fedora-Developer-38-20230519.n.0.SiFive.Unmatched.and.QEMU/Fedora-Developer-38-20230519.n.0-nvme.raw.img.xz
wget https://dl.fedoraproject.org/pub/alt/risc-v/disk_images/Fedora-Developer-38-20230519.n.0.SiFive.Unmatched.and.QEMU/Fedora-Developer-38-20230519.n.0-nvme.raw.img.xz.sha512sum

# Verify
sha512sum -c *.sha512sum

# Uncompress
unxz Fedora-Developer-38-20230519.n.0-nvme.raw.img.xz

# Download the latest U-Boot (we need U-Boot SPL and U-Boot ITB files)
mkdir u
pushd u
rpm2cpio http://fedora.riscv.rocks/kojifiles/packages/uboot-tools/2023.04/1.4.riscv64.fc38/noarch/uboot-images-riscv64-2023.04-1.4.riscv64.fc38.noarch.rpm | cpio -dvim
popd

# use kpartx:
mkdir -p /tmp/disk_img
sudo kpartx -a -v Fedora-Developer-38-20230519.n.0-nvme.raw.img
# Output example
add map loop2p1 (253:4): 0 1433600 linear 7:2 34
add map loop2p2 (253:5): 0 19537853 linear 7:2 1433634
# We need to mount the 1st partition (or /boot partition)
sudo mount /dev/mapper/loop2p1 /tmp/disk_img
# Edit extlinux.conf
sudo nvim /tmp/disk_img/extlinux/extlinux.conf

# Existing boot entry

 # Launch QEMU
qemu-system-riscv64 \
  -bios u/usr/share/uboot/qemu-riscv64_spl/u-boot-spl.bin \
  -nographic \
  -machine virt \
  -smp 4 \
  -m 8G \
  -device loader,file=u/usr/share/uboot/qemu-riscv64_spl/u-boot.itb,addr=0x80200000 \
  -object rng-random,filename=/dev/urandom,id=rng0 \
  -device virtio-rng-device,rng=rng0 \
  -device virtio-blk-device,drive=hd0 \
  -drive file=Fedora-Developer-38-20230519.n.0-nvme.raw.img,format=raw,id=hd0 \
  -device virtio-net-device,netdev=usernet \
  -netdev user,id=usernet,hostfwd=tcp::10000-:22
